# 如何分类现代编程方法？我对当代编程的理解

我想当然地认为，编程，编写程序，应当由人、键盘来完成。

在现在，我们会把如此称作「古法编程」。显然，AI 的出现带动了很多事物的改变，而 AI 的本源——程序，其相关更是那波动最大的一部分。我们常讲 “本是同根生，相煎何太急”，但事实如此，AI 正一步步地改写我们的生活方面。写作本文时，AI 所驱动的安全审计正在知名项目中搜寻漏洞并取得重大进展，我们必须感谢开源阵营、感谢愿意开放源代码的项目作者，感谢自由软件阵营、感谢 FSF（Free Software Foundation，自由软件基金会）和GNU，他们撑起了当今互联网的太多——我本身是一个开源精神的拥护者。

AI 真是个好东西，它把如今编程的门槛压得越来越低——也未免太低了。先前，我作了一文，讲了讲我对  Vibe Coding  相关的看法，那文所作，实在急促，因为是和人争论而兴怒作的，因此表述可能也存在不足之处。我以为我有必要先给那人道个歉，从争论开始时，篇幅、主语、语境、切入就有了些许的差异，而在后来对话被放大数倍了。我自然没有资格挑别人的理，但是我应当思考我的问题——对他人的指正没有合理反思。我们就不应当展开这个谈论，或许我就不应产生任何交集——这已经是哲学的本源问题了，不在本文所论之内。

借本文主题，我想谈论下我对各类编程模式的看法。

## 一、所谓「古法编程」

我们无法绕过这一点去直接谈论其他的，但也正合我意、再好不过了。向来，我一直认为，人在代码中永远是最重要的那个，一位程序员所必须满足的，是其应至少会几门计算机语言；如果他什么都不会，全是靠 AI 开发的，那他就不配叫作这个词，哪怕是「开发者」也不太符合——毕竟只有 AI 在开发。我自己也不能妄自菲薄地称作自己这个词，因为我尚未真正掌握任何一门语言。

一直以来，编程都是所谓“古法”、“手搓”代码的，直到现今才有所改变。我觉得，至少从 00 后开始，大部分学编程是为了兴趣的，而不是什么单纯为工作而学的——且我也不认为如此学得进去。在我看来，代码就能够算作艺术的一种，是如此「神圣」的。

 
## 二、所谓「半古法编程」

这是我近来新学的词，用来描述下文所讲真是再合适不过，只是这一切都是以新的“Vibe”式编程——也就是我将会在第三节讨论的事物——其现在的地位而定义，或许随着技术进一步地进步，名称和属性会有进一步变化。

这里所讲的，是指用应用层作开发的。通常来讲，是使用 WebUI 之类直接作 AI 辅助开发的方式。我认为，这种方式最合适的需求是以单个或少数个文件的快速修改为基点的辅助编程。我认为我的项目体量还未大到一定分量上，所以尚无使用更高一阶的必要，但这一问题我正在考虑。

如此的优点：

1. AI 没有操作文件的任何权限，真正做到人工审批落实。
2. 基于 ToC 端，免费额度高，使用花费在不同情况下不定但总体不多（未考虑服务提供商限制，大部分AI免费额度足够）。
3. 折腾代价小（搬运代码的人力成本不算）。

如此的缺点：
1. 不适合大型项目，这种方案我认为只适合有明确分点目标的小删小改，而不适用于单次大改。你要把自己作为「想偷懒的工程师」看待，而不是只管提要求的甲方。
2. 必须要熟悉项目，应该严格依靠“源代码 + 需求 -> AI 加工 -> 结果”的方式运作，所以如果你不想如此麻烦，请参看第三节。

## 三、 Vibe Coding  之类
让我们先查查词典： “vibe”，<非正式>n.氛围。顾名思义，这就是「氛围编程」——一种以 AI 为主体的编程方式。呵！可喜可贺，你终于可以扮演一名挑刺的甲方了。其和第二节所提内容最大的表层区别就是它可以自己读项目源代码和自己将代码写入文件了，当然我们不能如此表层地看待。

到达这一步，虽然你要做的配置变多了，但你能配置的也更多了。在此我不得不对那位人表示感谢，因为其对我以后转到这类方式和作本文的过程中作了铺垫。在以客户端（如 OpenCode、Claude Code、OpenAI Codex 等）+ AI（API 调用，来自服务商）的工作机制中，客户端和 AI 的选择性都极其丰富。不仅客户端形态各有千秋，AI 使用 API 来源也有极大空间。就以 AI 来看， Temperature 、 Top-p  等等都对使用有影响，这些参数共同决定了 AI 的风格，而客户端的 AI 调用工具是读取文件、写入文件、执行命令的基础。

但以下几点必须要注意：
1. 安全性。我永远不建议一个纯小白直接上手开发使用  Vibe Coding ，如果你想，之前有个叫“[灵光AI](https://www.lingguang.com/)”的热度挺高，不如去试试那个。近来越来越多的用户密钥泄露事件，多半是因为新手对代码的密钥管理等问题了解甚微，因此先学学 Git 基础、提交前再让 AI 帮你检查一下文件里有无泄露信息都是极其有必要的。
2. 价钱。不要忘记你用的是商用服务，如果你不想面对天价账单你最好提前做好规划，你的 API 可是明码标价的。
